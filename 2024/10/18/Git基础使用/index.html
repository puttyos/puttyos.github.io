Git基础使用

发表于 2024-10-18 本文字数： 1.9k 阅读时长 ≈ 7 分钟

Git的基础使用教程

1.使用方式

命令行

图形化界面（GUI）

IDE插件/拓展

2.初始化配置

配置名称：git config –global user.name “Yufei Tan”

省略（local）：本地配置，只对本地仓库有效

global：全局配置，对所有仓库有效

system：系统配置，对所有用户生效

配置邮箱：git config –global user.email 3518851990@qq.com

查看配置的信息：git config –global –list

3.创建仓库

mkdir learn-git 创建目录

git init 本地创建

git clone 远程克隆

4.工作区域和文件状态

本地数据管理：

工作区域（Working Directory）：.git所在目录

暂存区（Staging Area/Index）：保存即将提交到git仓库的修改内容

本地仓库(Local Repository)：包含完整的项目历史和元数据，Git储存代码和版本信息的主要位置

Git文件状态：

未跟踪：新创建的，未被git管理的文件

未修改、已修改、已暂存

5.添加和提交文件

git status 查看仓库状态 加-s查看简略模式，??表示暂存区、工作区的状态，M表示被修改过

git add file 提交文件到暂存区

（*.txt代表所有txt后缀的文件，.代表当前文件夹所有文件

git rm –cached file 取回文件

git commit -m”添加提交信息” 不指定-m的话会进入交互式界面，默认使用vim编辑提交信息

git commit -a -m（或-am）”添加提交信息”暂存提交一键进行

（按i进入编辑模式，esc退出，:wq保存退出）

git log查看提交记录（git log –oneline查看简洁的提交记录）

git ls-files查看暂存区

6.回退版本

git reset –soft +回退的版本ID/+HEAD^（回退到上一个版本）保留工作区、暂存区

HEAD指向分支的最新节点

git reset –hard 丢弃工作区、暂存区 +提交时间ID可以回复仓库状态

git reset –mixed(默认参数) 只保留工作区、丢弃暂存区

git reset –keep 保留工作区最新内容、丢弃暂存区

cp -rf repo repo1 复制目录

ctrl+n快速打开另一个终端

git reflog查看操作历史

扎到误操作之前的版本号，再使用git reset –hard+版本号回退

7.查看差异

git diff查看不同区/文件不同版本/文件在两个分支的差异

默认比较工作区和暂存区的内容

git diff HEAD比较工作区和版本库的差异

git diff –cached 比较暂存区和版本库的差异

git diff +两个版本的提交ID/ID和HEAD/HEAD~或HEAD^ (上一个版本) HEAD 比较版本的内容差异

HEAD~2表示前两个版本，若再+文件名，则只会查看特定文件的差异内容

vi指令进入vim编辑器

8.删除文件

ls -ltr 按时间顺序显示详细信息

直接删除 rm 文件名，只会删除本地库中的文件，不会删除暂存区中的，在用一次git add删除暂存库中的文件

git rm 文件名可以两个区同时删除，但还要commit到版本库中

9.忽略文件

gitignore

把文件添加到.gitignore文件中，该文件不能是已经被添加到库中的文件

git默认不把空的文件夹纳入版本管理

忽略文件夹的时候文件夹以斜线结尾

.gitignore文件的匹配规则：从上到下逐行匹配，每一行表示一个忽略模式

空行或以#开头的行会被忽略，一般空行用于可读性的分割，#用于注释

使用标准的Glob模式匹配，例如：

*通配任意字符，?匹配单个字符，[]表示匹配表中的单字符，如[abc]表示a/b/c

**匹配任意的中间目录，[a-z]表示任意一位小写字母，!表示取反

10.SSH配置和克隆仓库

HTTPS需要验证账户密码，SSH不需要，但需要添加公钥配置

cd 回到根目录

cd .ssh

ssh-keygen -t rsa -b 4096如果已设置过就要新建一个文件，并且配置github，没有就一直回车

进入公钥文件复制公钥到GitHub上

id_rsa是私钥不能给，有.pub 是公钥上传到GitHub

git clone +SSH地址

git push把本地修改推送给远程仓库，容易报错，改为git push -u origin main（这边注意新分支的名称）

git pull把远程仓库的修改拉取到本地仓库

11.关联本地仓库和远程仓库

git remote add

第一种如上图中的提示：git push –set-upstream origin master。其中的origin是你在clone远程代码时，git为你创建的指向这个远程代码库的标签，它指向repository。为了能清楚了解你要指向的repository，可以用命令git remote -v进行查看。master是你远程的branch，可以用git branch -a查看所有分支，远程分支是红色的部分。然后确定好这两个值后，将值换掉即可。
另一种方法是：git push -u origin master。同样根据自己的需要，替换origin和master。
两个命令的区别是第一条命令是要保证你的远程分支存在，如果不存在，也就无法进行关联。而第二条指令即使远程没有你要关联的分支，它也会自动创建一个出来，以实现关联。

git remote -v 查看远程仓库地址

git branch -m master main 指定新分支名称为main

导致错误的原因就是没有理解 origin 和 master

clone 操作之后，Git 会自动将此远程仓库命名为 origin ，建立指针指向项目的指针master

origin只相当于一个指针指向你 github 远端的地址（仓库名）/（分支名）origin/master 代表远程分支

git pull 远程仓库 远程分支:本地分支

注意：拉取的时候会自动执行合并操作，如果不冲突就成功，冲突就要手动解决

还可以用 git fetch 区别在于只会获取远程仓库的修改，不会合并

14.分支

代码库中的不同版本

独立开发，持续集成，提高协作效率

git branch 查看当前所有分支

git branch +名字 创建新分支

git checkout +名字 切换到分支，但还可以恢复修改的文件名，但会有歧义

git switch +名字 切换分支

未合并之前旧分支不包含新分支

切换到main分支使用git merge +名字 把分支合并到main分支

git log –graph –oneline –decorate -all 查看分支图

alias定义别名 alias graph=”git log –graph –oneline –decorate -all”

合并后分支还是在，手动删除使用git branch -d +名字 删除已合并的分支 +D强制删除

冲突需要手动合并或者使用git merge –abort中断合并

rebase变基，找到共同祖先再把之后新的变化转接到另一个分支上

git checkout -b +文件名 +提交时间ID 回复到当前状态
